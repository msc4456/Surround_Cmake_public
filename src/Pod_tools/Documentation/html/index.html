Overview

PVRTools is a collection of source code to help developers with some common tasks which are frequently used in 3D programming. PVRTools supplies code for mathematical operations, matrix handling, loading 3D models and to optimise geometry. Sections which are specific to certain APIs contain code for displaying text and loading textures.

File formats

The following file formats are used in PVRTools:

PFX format

PFX (PowerVR Effects) files are used to store graphics effects. As a minimum, a correctly formatted PFX consists of:

  • One EFFECT block
  • One VERTEXSHADER block
  • One FRAGMENTSHADER block

It is also possible for PFXs to contain the following:

  • One TARGET block
  • Zero or more TEXTURE blocks

By default PFXs are stored in .pfx files. It is possible for multiple PFXs to exist within a single .pfx file, each described by a separate effect block; in this instance multiple PFXs may share blocks. Finally, it is possible for a PFX to reference a TARGET block as an input as if it were a TEXTURE block, enabling the simple creation of complex post-processing effects. For this to function correctly the TARGET block render should be completed prior to being read as an input. If the TARGET block render has not been completed prior to being read as an input, the behaviour will vary based on the render target implementation of the platform.

For more information see the PFX File Format Specification.

POD format

POD files store data for representing 3D scenes; including geometry information, animations, matrices, materials, skinning data, lights, cameras, and in some instances custom meta-data. These files are output by the PVRGeoPOD tool, and are designed for deployment through optimistions such as triangle/vertex sorting and data stripping.

The format is designed to be easily read, for information on the required algorithm and the overall structure of the format see the POD File Format Specification.

PVR format

PVR files are used as a container to store texture data. PVR files can be exported from PVRTexTool and a number of third party applications.

For more information see the PVR File Format Specification.

Header files

Here is a list of common header files present in PVRTools:

  • PVRTArray.h: A dynamic, resizable template class.
  • PVRTBackground.h: Create a textured background.
  • PVRTBoneBatch.h: Group vertices per bones to allow skinning when the maximum number of bones is limited.
  • PVRTDecompress.h: Descompress PVRTC texture format.
  • PVRTError.h: Error codes and tools output debug.
  • PVRTFixedPoint.h: Fast fixed point mathematical functions.
  • PVRTGlobal.h: Global defines and typedefs.
  • PVRTHash.h: A simple hash class which uses TEA to hash a string or given data into a 32-bit unsigned int.
  • PVRTMap.h: A dynamic, expanding templated map class.
  • PVRTMatrix.h: Vector and Matrix functions.
  • PVRTMemoryFileSystem.h: Memory file system for resource files.
  • PVRTMisc.h: Skybox, line plane intersection code, etc...
  • PVRTModelPOD.h: Load geometry and animation from a POD file.
  • PVRTPFXParser.h: Code to parse our PFX file format. Note, not used in fixed function APIs, such as OpenGL ES 1.x.
  • PVRTPrint3D.h: Display text/logos on the screen.
  • PVRTQuaternion.h: Quaternion functions.
  • PVRTResourceFile.h: The tools code for loading files included using FileWrap.
  • PVRTShadowVol.h: Tools code for creating shadow volumes.
  • PVRTSkipGraph.h: A "tree-like" structure for storing data which, unlike a tree, can reference any other node.
  • PVRTString.h: A string class.
  • PVRTTexture.h: Load textures from resources, BMP or PVR files.
  • PVRTTrans.h: Transformation and projection functions.
  • PVRTTriStrip.h: Geometry optimization using strips.
  • PVRTVector.h: Vector and Matrix functions that are gradually replacing PVRTMatrix.
  • PVRTVertex.h: Vertex order optimisation for 3D acceleration.
